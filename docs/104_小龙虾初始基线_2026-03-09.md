# 小龙虾初始基线 v1.0

## 这次真正定下来的标准
以后每次新建一个小龙虾，默认不是“随便装一堆能力”，而是统一按 **初始虾 1.0** 出生。

这个标准只有 5 件事：
- 纯净 OpenClaw
- QMD
- 默认 22 个 skill
- 自动发文件
- 不预设角色身份

也就是说，新虾一出生就应该是“能聊、能搜、能记、能干活、能把文件发回来”的通用助手，
而不是一上来就被写死成 CEO、CTO、写手、分析师。

## 初始虾 1.0 的出生配置
1. OpenClaw 固定稳定版本
2. 独立目录、独立端口、独立 service
3. 一个确认可用的模型
4. 飞书长连接
5. QMD 记忆
6. `tools.profile=full`
7. 中性 workspace 文件骨架
8. 默认 22 个 skill
9. 文件发送规则默认开启

## 以后怎么用
- 新建小龙虾：直接按这套 1.0 基线装
- 有额外需求：再按用户任务去加 skill
- 本仓库没有的 skill：先查我们自己的 Git 仓库，再去外部 skill 网站找
- 角色身份：后面由主人自己定义，不在出生时强塞

## 一句话结论
以后“新虾默认配置”这件事，到这里算正式固定：

**纯净 OpenClaw + QMD + 22 个默认 skill + 自动发文件 + 不预设角色。**
