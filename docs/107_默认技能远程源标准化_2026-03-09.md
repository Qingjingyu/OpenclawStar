# 107 默认技能远程源标准化 2026-03-09

## 本次目标
把默认出生包的 skill 安装从“依赖本机碰巧已有 skill”升级成“本地优先，远程兜底”。

## 为什么要做
之前 `install.sh` 虽然能把新虾骨架搭起来，但默认 19 个 skill 里，大部分要看机器上有没有现成 skill 源。
这会导致同一个仓库，在不同机器上的结果不一样。

说人话：
- 以前像是有菜谱但食材常常不在厨房里
- 现在至少把食材采购地址也写进去了

## 这次落地了什么
1. 新增统一来源清单：`bootstrap/skill_sources.sh`
2. `install_base_skills.sh` 改成：
   - 先找本机 skill
   - 找不到再去远程 URL 拉取
3. `install_finance_skills.sh` 也改成走同一套来源清单

## 已确认的远程模式
当前默认技能和金融技能都可以用：
- `https://cmcm.bot/skills/<skill-name>.md`

本次已逐个验证：
- 默认 19 个 skill
- `us-stock-analysis`

## 结果
现在 `OpenclawStar/install.sh` 的行为更稳定：
- 能自动铺骨架
- 能自动补默认 skill 的远程来源
- 如果未来某个 skill 源失效，会明确报错，不会假装成功

## 意义
这一步之后，`OpenclawStar` 更像真正的“新虾母版仓库”了，而不只是一个说明仓库。
