# 108 Skill 策展仓库 2026-03-09

## 本次决定
OpenclawStar 不再只做“出生模板仓库”，还要同时承担“技能策展仓库”的角色。

## 核心思路
默认新虾不是 CEO、CTO、RD。
默认新虾首先是一只通用 AI 助手。

也就是说：
- 先有一个通用助理底座
- 再按需要给它装 skill
- 真有复杂团队需求时，再做多角色分工

## 这次落地了什么
1. 新增 `skills/` 目录
2. 按分类放入当前默认技能与金融技能
3. 新增 `skills/INDEX.md`
4. 新增 `skills/external-sources/README.md`

## 当前分类
- `skills/search/`
- `skills/growth/`
- `skills/coordination/`
- `skills/automation/`
- `skills/governance/`
- `skills/system/`
- `skills/finance/`
- `skills/external-sources/`

## 意义
以后新建一只虾时，思路会更清楚：
- OpenclawStar 负责把虾生出来
- skills 目录负责教它本事

这样就不会先陷入 CEO/CTO/工程师这些过重的身份设定，而是先让它成为一只真正能帮人办事的通用助手。
